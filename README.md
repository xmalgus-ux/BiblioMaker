# BiblioMaker

BiblioMaker - настольное приложение для оформления библиографических списков по ГОСТ Р 7.0.100-2018. Программа принимает список литературы вручную или извлекает его из Word-документа `.docx`, отправляет записи на обработку выбранному ИИ-агенту и позволяет сохранить исправленный результат.

## Возможности

- ручной ввод библиографических записей;
- извлечение списка литературы из `.docx`;
- замена списка литературы в исходном Word-документе с сохранением результата в новый файл;
- сохранение результата в `.txt` или `.docx`;
- автоматическая эвристическая классификация источников;
- уточнение типа источника и недостающих данных перед обработкой;
- поддержка ИИ-агентов `YandexGPT` и `GigaChat`;
- история обработок в SQLite;
- просмотр логов ответов ИИ;
- светлая и темная темы;
- интерфейс на русском, английском и китайском языках.

## Структура проекта

```text
gost-assistant/
  main.py                     # точка входа приложения
  app/
    core/
      ai_providers.py         # ИИ-агенты: YandexGPT и GigaChat
      database.py             # SQLite: история и настройки
      docx_processor.py       # чтение и сохранение Word-документов
      source_classifier.py    # классификация типов источников
    gui/
      main_window.py          # PyQt6-интерфейс
  config/
    settings.json             # базовые настройки
  data/
    gost_assistant.db         # локальная БД, создается/используется приложением
    logs/                     # логи ответов и ошибок ИИ
```

## Требования

- Python 3.11+;
- Windows, Linux или macOS с поддержкой PyQt6;
- доступ к интернету для работы с `YandexGPT`.

Основные Python-зависимости:

```text
PyQt6
python-docx
requests
python-dotenv
```

## Установка

Перейдите в каталог приложения:

```powershell
cd D:\PythonProject\GOSTTransformerQwen\gost-assistant
```

Создайте и активируйте виртуальное окружение:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

Установите зависимости:

```powershell
pip install PyQt6 python-docx requests python-dotenv
```

## Настройка YandexGPT

Для реальной обработки через YandexGPT откройте в приложении `Настройки` -> `Настройка ИИ-агента`, выберите `YandexGPT` и укажите:

- `YANDEX_API_KEY`;
- `YANDEX_FOLDER_ID`.

Значения сохраняются локально в SQLite-базе настроек приложения в зашифрованном виде. На Windows используется DPAPI с привязкой к текущему пользователю.

Также поддерживается запасной вариант через файл `.env` в каталоге `gost-assistant`:

```env
YANDEX_API_KEY=ваш_api_key
YANDEX_FOLDER_ID=ваш_folder_id
```

Файл `.env` содержит секреты и не должен попадать в Git. При распространении приложения не кладите в сборку свой личный API-ключ.

## Настройка GigaChat

Для GigaChat пользователь вводит готовый `Authorization Key` самостоятельно в приложении:

```text
Настройки -> Настройка ИИ-агента -> GigaChat
```

В этом же окне есть кнопка `Как получить Authorization Key` с краткой инструкцией:

1. Перейдите по ссылке https://developers.sber.ru/portal/products/gigachat-api и зарегистрируйтесь.
2. Перейдите в личный кабинет.
3. Создайте новый проект с GigaChat API или выберите существующий проект.
4. Перейдите в созданный проект и нажмите кнопку `Настроить API`.
5. Сгенерируйте `Authorization Key`, если он еще не создан.
6. Скопируйте значение поля `Authorization Key`.

Введенное значение сохраняется локально в SQLite-базе в зашифрованном виде.

Scope для запроса к GigaChat задан в коде как `GIGACHAT_API_PERS`.

При первом запросе к GigaChat приложение автоматически подготовит локальный CA-bundle в `data/certs`: скачает корневой сертификат Минцифры и добавит его к стандартным сертификатам `certifi`. Это нужно, чтобы Python мог проверить SSL-сертификаты GigaChat API.

## Запуск

Из каталога `gost-assistant` выполните:

```powershell
python main.py
```

После запуска откроется окно `BiblioMaker - Оформление по ГОСТ`.

## Сборка в EXE

Для сборки установите PyInstaller:

```powershell
pip install pyinstaller
```

Сборка папкой:

```powershell
pyinstaller --noconfirm --windowed --name BiblioMaker `
  --add-data "config;config" `
  --add-data ".env;." `
  main.py
```

Готовый файл будет в:

```text
dist\BiblioMaker\BiblioMaker.exe
```

Сборка одним exe-файлом:

```powershell
pyinstaller --noconfirm --onefile --windowed --name BiblioMaker `
  --add-data "config;config" `
  --add-data ".env;." `
  main.py
```

Встроенный `.env` будет доступен приложению при первом запуске. Приложение прочитает `YANDEX_API_KEY` и `YANDEX_FOLDER_ID`, после чего сохранит их в локальную SQLite-базу в зашифрованном виде.

Важно: встроенный `.env` все равно можно извлечь из сборки техническими средствами. Это удобно для распространения внутри доверенного контура, но не является полноценной защитой секрета.

## Как пользоваться

1. Выберите источник данных: ручной ввод или файл `.docx`.
2. Если выбран ручной ввод, вставьте список литературы, каждый источник с новой строки.
3. Если выбран `.docx`, нажмите `Открыть документ` и выберите Word-файл.
4. Выберите ИИ-агента:
   - `yandex` - обработка через YandexGPT.
   - `gigachat` - обработка через GigaChat.
5. Нажмите `Исправить по ГОСТ`.
6. При необходимости уточните типы источников и недостающие данные в открывшемся окне.
7. Проверьте исправленный список.
8. Нажмите `Сохранить` и выберите формат результата.

## Работа с Word-документами

Приложение ищет раздел библиографии по заголовкам:

- `Список литературы`;
- `Библиографический список`;
- `Использованные источники`;
- `Литература`;
- `References`.

Если раздел найден, старые записи заменяются исправленными. Если раздел не найден, приложение может добавить новый раздел `Список литературы` в конец документа.

При включенной настройке резервного копирования перед сохранением создается копия исходного `.docx` с суффиксом вида:

```text
_backup_YYYYMMDD_HHMMSS.docx
```

## Данные и логи

- История обработок хранится в `gost-assistant/data/gost_assistant.db`.
- Сырые ответы ИИ записываются в `gost-assistant/data/logs/ai_raw_responses.log`.
- Ошибки ИИ записываются в `gost-assistant/data/logs/ai_errors.log`.

Каталог `data` содержит локальные пользовательские данные. Файлы БД и логов не стоит коммитить в репозиторий.

## Настройки

В окне настроек можно изменить:

- язык интерфейса;

- ИИ-агента по умолчанию;

## Ограничения

- Классификация источников основана на эвристиках и может требовать ручного уточнения.
- YandexGPT не должен придумывать недостающие библиографические данные, но итоговый результат все равно нужно проверять.
- Для корректной обработки `.docx` желательно, чтобы раздел списка литературы имел один из поддерживаемых заголовков.
